# gRPC S2S MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a typed Server1 -> Server2 gRPC API for invoking `zeroclaw.exe` agent runs.

**Architecture:** Implement the gRPC binding inside `crates/zeroclaw-gateway` with an in-memory run registry. The service reuses `PairingGuard`, `SessionActorQueue`, SQLite session persistence, and `Agent::turn_streamed`; CLI startup is exposed as `zeroclaw grpc`.

**Tech Stack:** Rust 2024, tonic 0.14, prost 0.14, tokio, ZeroClaw gateway/runtime crates.

---

### Task 1: Protobuf Contract

**Files:**
- Create: `crates/zeroclaw-gateway/proto/zeroclaw/v1/agent.proto`

- [ ] **Step 1: Write the protocol file**

Define `zeroclaw.v1.AgentService` with `CreateRun`, `StreamRun`, `CancelRun`, and `GetRun`. Include request identity (`protocol`, `request_id`, `session_id`, `actor`), input (`RunInput`), options (`RunOptions`), metadata map, stable run state, and typed event payloads for accepted, started, deltas, tool calls/results, completed, cancelled, and failed.

- [ ] **Step 2: Verify field stability**

Use reserved-compatible field numbering: keep identifiers at low field numbers, maps/options after core fields, and typed event payloads inside a `oneof`.

### Task 2: Gateway gRPC Module Tests

**Files:**
- Create: `crates/zeroclaw-gateway/src/grpc.rs`
- Modify: `crates/zeroclaw-gateway/src/lib.rs`

- [ ] **Step 1: Add failing unit tests**

Add tests for:
- request validation rejects missing `request_id`, missing `session_id`, and empty text;
- auth accepts disabled pairing, accepts valid bearer token, and rejects invalid bearer token;
- duplicate `request_id` for the same authenticated caller returns the same `run_id`;
- unknown `CancelRun` is idempotent and returns `accepted = false`;
- `TurnEvent` maps to protocol events with the correct event type.

- [ ] **Step 2: Run the targeted tests**

Run: `cargo test -p zeroclaw-gateway grpc`

Expected: tests fail because the gRPC module is not implemented yet.

### Task 3: gRPC Runtime Implementation

**Files:**
- Modify: `crates/zeroclaw-gateway/Cargo.toml`
- Modify: `crates/zeroclaw-gateway/src/grpc.rs`
- Modify: `crates/zeroclaw-gateway/src/lib.rs`

- [ ] **Step 1: Add dependencies**

Add direct dependencies for `tonic`, `tonic-prost`, `prost`, `prost-types`, `bytes`, `http`, `http-body`, and `tower-service`.

- [ ] **Step 2: Implement prost message types and tonic server glue**

Manually define the generated-equivalent Rust protocol module so builds do not require a system `protoc`. Keep `agent.proto` as the source-of-truth contract for clients.

- [ ] **Step 3: Implement `GrpcAgentService`**

Implement:
- bearer metadata authentication using `authorization: Bearer <token>`;
- validation for request/session/text;
- process-local idempotency keyed by authenticated caller plus `request_id`;
- retained event replay with a 1,024 event cap;
- run cancellation with `tokio_util::sync::CancellationToken`;
- terminal run status for `GetRun`.

- [ ] **Step 4: Implement background execution**

For each new run:
- acquire `SessionActorQueue` on `gw_<session_id>`;
- load SQLite-backed session history when enabled;
- seed `Agent::from_config`;
- call `turn_streamed`;
- map `TurnEvent` into `RunEvent`;
- persist user and assistant messages;
- emit `run.completed`, `run.cancelled`, or `run.failed`.

### Task 4: CLI Wiring

**Files:**
- Modify: `src/lib.rs`
- Modify: `src/main.rs`
- Modify: `Cargo.toml`

- [ ] **Step 1: Add command shape**

Add `Grpc { port: Option<u16>, host: Option<String>, project_root: PathBuf }` to the CLI. Default host is `config.gateway.host`; default port is `config.gateway.port + 1` with saturating fallback to `42618`.

- [ ] **Step 2: Wire startup**

Apply project root to config, then call `zeroclaw_gateway::grpc::run_grpc_server(&host, port, config)`.

- [ ] **Step 3: Preserve security warning behavior**

Use the same public-bind warning policy as the HTTP gateway.
